# Encore Migration Plan

RecruitProof should not lead with replacement. Lead with proof.

## Six phases

1. Read-only export
2. Parse and normalize
3. Dedupe and repair
4. Index and benchmark
5. Shadow recruiter workflow
6. Cutover, archive-only mode, or hybrid mode

## Rollback

Because the first proof is read-only, rollback is simply deleting generated artifacts and source export copies.
