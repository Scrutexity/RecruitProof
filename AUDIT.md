**Where it appears:** `proof/million_cv_scan/sample_report.md`, `PERFORMANCE.md`

**Status:** FABRICATED. No recruiter has ever reviewed RecruitProof output.
**Where it appears:** `proof/million_cv_scan/sample_report.md`, `SAMPLE_OUTPUT.md`
**Fix:** Remove. Replace with: "RecruitProof surfaces candidates based on semantic + skill matching. Whether they're 'hidden' depends on your team's existing shortlist."

### ⚠️ "87% savings vs Eightfold, 95% vs LinkedIn Recruiter"

**Status:** These are mathematically derived from the pricing assumptions in `competitive_intel.py`. The competitor prices are real (from public sources). **Where it appears:** `BUSINESS.md`, `competitive_intel.py`, the Next.js dashboard

**Status:** PARTIALLY TRUE. The architecture is GDPR-compatible (local-first, right to erasure, data portability). But "compliant" implies a legal review has been completed. It hasn't.

## What's Partially Implemented

### ⚠️ Next.js Dashboard → API Wiring

The Next.js dashboard has real API routes (`/api/import/start`, `/api/import/status/[job_id]`, `/api/proof/data`) that DO work — they read real ZIP filenames, process them, and return real evidence. But the Import Center is the only section fully wired to real data. The other sections (Executive, Intelligence, Search, Candidate, ROI, Migration, Audit, Trust) use hardcoded mock data from `src/lib/demo/data.ts`. This is acceptable for a sales demo but not for production.

### ⚠️ LLM Reasoning (Ollama/OpenAI)

The `agents.py` module has a `LocalLLMClient` that auto-detects Ollama at `localhost:11434`. The code is real and will work if Ollama is running. But it has not been tested with a live Ollama instance (none was available in the sandbox). The OpenAI path is also implemented but untested (no API key). The rule-based fallback IS tested and works.
